# 해변 예보 비교

여행과 해수욕 전에 여러 예보를 한 화면에서 비교하는 모바일 우선 웹앱입니다. ECMWF, GFS, ICON 수치모델을 각각 보여주고, Open-Meteo Marine으로 파고와 해양 상태를 함께 확인합니다.

## 주요 기능

- 기본 즐겨찾기: 아야진해수욕장, 공현진1리해수욕장, 화진포해수욕장, 마차진해수욕장, 대진1리해수욕장, 천진해수욕장, 금호설악리조트
- 지도 보기: Leaflet + OpenStreetMap 타일, 즐겨찾기 마커 선택, 지도 클릭 위치 조회
- 장소 검색: Open-Meteo Geocoding + OpenStreetMap Nominatim
- 기간 선택: 시작 날짜와 종료 날짜를 지정하고 해당 범위의 시간별 예보를 조회
- 출처별 시간표: KMA, ECMWF, GFS, ICON, AccuWeather, 해양예보를 탭으로 선택하고 가로 시간축으로 표시
- 모델 비교: ECMWF, GFS, ICON 시간별 강수량, 강수확률, 기온, 풍속, 돌풍, 구름량
- 모델 합의도: 세 수치모델만으로 강수 여부, 시작시각, 강수량 편차를 계산
- 해양 예보: 파고, 파주기, 파향, 너울, 수온, 해류, 육상 풍속/돌풍 병행 표시
- 물놀이 시간 판단: 특보, 강수, 뇌우, 풍속, 돌풍, 파고, 일몰 전 시간대를 기준으로 2~4시간 단위 표시
- 예보 변화 추적: localStorage에 이전 조회 결과를 저장해 강수량, 비 시작시각, 파고, 돌풍 변화를 비교
- PWA: `manifest.json`, `service-worker.js` 포함

## 실행

```bash
npm run dev
```

브라우저에서 `http://localhost:3000`을 엽니다.

이 프로젝트는 별도 패키지 설치 없이 Node.js 기본 기능과 브라우저 CDN(Leaflet, lucide)을 사용합니다.

## 환경변수

필수 키 없이도 Open-Meteo 수치모델과 해양 예보는 작동합니다. 로컬에서는 앱의 `설정 > 기상청 키`에 공공데이터포털 서비스키를 붙여넣으면 KMA 탭이 바로 활성화됩니다.

서버나 배포 환경에서는 아래 환경변수를 사용할 수 있습니다. 로컬 서버는 `.env` 파일도 자동으로 읽습니다.

```bash
KMA_API_KEY=
ACCUWEATHER_API_KEY=
```

- `KMA_API_PROVIDER`: `apihub` 또는 `data.go.kr`. API허브 키를 받은 경우 `apihub`를 사용합니다.
- `KMA_API_KEY`: 기상청 단기예보와 기상특보 조회
- `ACCUWEATHER_API_KEY`: AccuWeather 시간별 예보 조회

기상청 API허브 키는 계정 키와 별도로 사용할 API의 활용신청이 필요할 수 있습니다. KMA 탭이 “활용신청 필요”로 보이면 API허브에서 `예특보 > 단기예보 > 동네예보 조회 > 단기예보조회` 활용신청을 완료한 뒤 새로고침하세요.

키가 없거나 특정 API 호출이 실패해도 앱 전체는 멈추지 않고, 해당 서비스 카드만 비활성 또는 오류 상태로 표시합니다.

## 배포

Vercel에서 바로 배포할 수 있습니다.

```bash
npx.cmd vercel --prod
```

Vercel 프로젝트 설정의 Environment Variables에 필요한 키를 등록합니다. 자세한 순서는 `DEPLOY.md`를 참고하세요.

## 데이터와 단위

- 시간대: `Asia/Seoul`
- 기온: `℃`
- 풍속·돌풍: `m/s`
- 파고: `m`
- 파주기: `초`
- 강수량: `mm`

기본 해수욕장 좌표와 KMA 격자값은 기상청 전국해수욕장 위경도 자료를 기준으로 넣었습니다. 금호설악리조트 좌표는 OpenStreetMap Nominatim 결과를 사용했습니다.

## 주의 문구

이 앱은 예보 비교 도구이며 현장 안전을 보장하지 않습니다. 기상특보, 해변 입수통제, 안전요원의 안내를 우선하세요. 이안류는 일반 파고 데이터만으로 판정할 수 없습니다.
